يعمل نظام إدارة المحتوى Pages (Pages CMS) من خلال **تحرير ملفات مستودع GitHub مباشرةً** دون الحاجة إلى قاعدة بيانات منفصلة.

في هذا المشروع، يربط `.pages.yml` الموجود في الجذر بين الأمرين التاليين:

## 1. مجموعة المنشورات

يمكنك إنشاء مقالات جديدة وتعديل المقالات الموجودة عن طريق تسجيل مجلد `src/content/posts` كمجموعة.  
تتعامل واجهة المستخدم مع الحقول مثل `title` و `description` و `date`، `cover`، `body`، وغيرها من الحقول في واجهة المستخدم، وعند الحفظ يتم إرسال ملف Markdown الفعلي.

## 2. ملف إعدادات الموقع

يمكن ربط `src/content/site.json` بمحرر ملفات منفصل لتعديل المعلومات الوصفية مثل اسم المدونة والوصف وعنوان URL من خلال نظام إدارة المحتوى (CMS).

## تحميل الصور

مسار التحميل هو `static/uploads`.  
يتم حفظ صور النص الرئيسي في المسار `/uploads/...`، ويمكن الوصول إليها في GitHub Pages بعد الإنشاء عبر نفس المسار.

## طريقة التشغيل

1. قم بتعديل المقالات في نظام إدارة المحتوى (CMS) الخاص بـ Pages.
2. يتم تثبيت التغييرات في المستودع.
3. عند الدفع إلى فرع `main`، تقوم GitHub Actions بإعادة البناء.
4. يتم نشر أحدث النتائج الثابتة على GitHub Pages.

باختصار، هذه هي بنية تحافظ على تجربة المحرر مثل CMS بينما يظل "مصدر الحقيقة" الفعلي في Git.
