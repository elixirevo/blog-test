# Elixir Evo Journal

SvelteKit ベースの静的ブログテンプレートです。`@sveltejs/adapter-static` を使用してサイト全体をプリレンダリングし、GitHub Actions を通じて GitHub Pages にデプロイし、Pages CMS で Markdown コンテンツを編集できるように構成されています。

## スタック

- SvelteKit + `@sveltejs/adapter-static`
- `.github/workflows/deploy.yml` 経由の GitHub Pages デプロイ
- root ディレクトリの `.pages.yml` 経由の Pages CMS
- `src/content/posts` 内の Markdown 投稿
- DeepLの翻訳を`src/content/translations/<locale>/posts`に同期

## ローカル開発

```sh
bun install
bun run dev
```

`Pagefind` の検索インデックスは、静的ビルド後に生成されます。検索機能まで確認するには、以下の手順を実行してください。

```sh
bun run build
bun run preview
```

登録済みの翻訳言語ファイルまでローカルで更新するには、DeepL APIキーを入力し、以下のコマンドを実行してください。

```sh
DEEPL_API_KEY=your-key bun run translate:posts
```

## コンテンツ構造

- `src/content/site.json`: サイトのタイトル、説明、URL などのメタ情報
- `src/content/posts/*.md`: フロントマター + Markdown本文
- `src/content/translations/<locale>/posts/*.md`: DeepLが生成し、コミットする言語ごとの翻訳
- `static/uploads`: CMSにアップロードされた画像

投稿のフロントマターには、カテゴリとは別に複数のタグを設定できます。タグは`#`なしで保存され、画面上でも`dev`のように表示されます。

Pages CMSでは、`Tags`コレクションでタグを先に作成し、投稿編集画面の`Tags`フィールドで複数選択します。

```yaml
category: Technical
tags:
  - dev
  - git
```

1行の文字列形式も可能です。

```yaml
tags: 'dev, git'
```
